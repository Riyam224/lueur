## Summary

Extend the existing onboarding screen (3 pages, wave blob layout, PageView navigation)
with: (1) three custom CustomPainter illustrations replacing the current icon+glow
approach, (2) updated copy and typography per design spec, (3) refined spacing and
progress dots, (4) hide Skip on last page, and (5) a one-time show-once flag persisted
via Hive, read by the splash screen to bypass onboarding on subsequent launches.

## Extension: Luna SVG Animated Illustration (`onboarding_screen_2.dart`)

**Scope**: New file only — does not modify any existing onboarding files.

**Input**: "Replace the chat bubble illustration with the Luna SVG asset, add float + blink animations."

### Summary

Create `onboarding_screen_2.dart`, a new alternative onboarding screen that replaces the
`OnboardingChatPainter` (page 2 illustration) with an animated Luna SVG. The screen is
self-contained: all text, colors, button layout, and navigation remain identical to the
original screen. Only the illustration widget changes.

### Technical Context

**New dependencies**: None — `flutter_svg ^2.3.0` already in `pubspec.yaml`.

**Asset path**: `assets/illustrations/luna.svg` (registered under `assets/illustrations/`
in `pubspec.yaml`). The path `assets/images/luna.svg` does **not** exist — use the
correct path throughout.

**SVG characteristics**: `luna.svg` is a 1 024 × 1 024 viewBox complex illustration
(~1 MB, rasterised-style paths). It has **no named layers** — SVG layer-visibility
toggling is not possible at runtime via `flutter_svg`.

**Animations needed**:
1. **Float** — gentle vertical oscillation, ±8 px, 3 s cycle, easeInOut, infinite.
2. **Blink** — eyes swap from open to closed for 150 ms, triggered randomly every
   2.5–4 s. Requires a separate mechanism (see research.md Decision 6).

**Blink implementation decision**: Use a `CustomPainter` overlay (`LunaBlinkOverlay`)
that draws two short filled arcs over the SVG at the eye positions. An `AnimatedSwitcher`
cross-fades between the SVG alone and a `Stack` of SVG + overlay. Eye positions are
calibrated to the 180 × 180 rendered size (see contracts/luna_animation_widget_contract.md).

**Controllers**: Both controllers (`_floatController`, `_blinkTimer`) live in
`OnboardingLunaAnimated`'s own `State` — not in `onboarding_screen_2.dart`. The screen
stays stateless for the illustration concern.

### Complexity Tracking (Extension)

| Decision | Chosen | Justification |
|---|---|---|
| Blink without named SVG layers | `CustomPainter` overlay via `AnimatedSwitcher` | No second SVG exists; overlay keeps single-asset approach with no new dependencies |
| Float driver | `AnimationController` + `CurvedAnimation(Curves.easeInOut)` | Standard Flutter pattern; `repeat(reverse: true)` gives natural oscillation |
| Blink timer | `dart:async Timer` (not another `AnimationController`) | One-shot random-interval trigger is simpler than a looping controller for this pattern |
| Widget extraction | `OnboardingLunaAnimated` in its own file | Keeps `onboarding_screen_2.dart` lean; widget is independently testable |
